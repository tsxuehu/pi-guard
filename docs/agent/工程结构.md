# Agent 工程结构

`project/agent` 按能力域分目录，**每个叶子目录是独立静态库**（各自 `CMakeLists.txt`）。

## 目录

| 路径 | 说明 |
|------|------|
| `src/foundation` | 基础层，`pi_guard_foundation`（INTERFACE），头文件 `include/foundation/`，`piguard::foundation` |
| `src/modules/...` | 采集 / 处理 / 输出 / infra / external_access，子目录一库一 `include/`，与实现同层 |
| `src/runtime` | 编排，实现 `runtime/app/`，头 `runtime/include/` |
| `src/main.cpp` | 入口 |
| `include/` | 可执行 `pi-guard-agent` 的附加私有头（按需） |
| `tests/` | `BUILD_TESTING` 时加入 |
| 根 `CMakeLists.txt` | 串联子目录，产物 `pi-guard-agent` 链接 `pi_guard_runtime` |

**头文件：** 各库 `include/` 下再分子目录，避免与短名冲突。例：`#include "capture_audio/audio_capture.hpp"`。`runtime` **PUBLIC** 链上各子库，依赖方自动带齐头路径。

## 构建

```bash
cd project/agent
cmake -B build -S . && cmake --build build
```

可执行文件：`pi-guard-agent`。构建目录勿提交。
