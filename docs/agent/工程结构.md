# Agent 工程结构

`project/agent` 采用三层结构：

- `src/modules/`：五大核心模块
  - `capture`
  - `processing`
  - `output`
  - `infra`
  - `external_access`
- `src/runtime/`：运行编排层（实现位于 `runtime/app`，公开头位于 `runtime/public`）
- `src/foundation/`：基础能力层（公开头位于 `foundation/public`，命名空间为 `piguard::foundation`）

## 目录重点

- `src/main.cpp`：程序入口
- `src/modules/<module>/public/`：各核心模块统一的对外头文件出口
- `src/modules/<module>/*/`：模块内部实现目录（通常 `*.cpp`）
- `include/piguard/`：公共头文件
- `CMakeLists.txt`：主构建入口

## 构建方式

采用“模块静态库 + 最终可执行程序”：

- `foundation`：`pi_guard_foundation`（INTERFACE）
- 每个核心模块各自 `CMakeLists.txt`，编译为静态库：
  - `pi_guard_capture`
  - `pi_guard_processing`
  - `pi_guard_output`
  - `pi_guard_infra`
  - `pi_guard_external_access`
- `runtime`：`pi_guard_runtime`（静态库，包含 `runtime/app/agent_app.cpp`，公开头在 `runtime/public`）
- 主 `CMakeLists.txt` 负责：
  - `add_subdirectory(src/foundation)`、`add_subdirectory(src/modules/...)`、`add_subdirectory(src/runtime)`
  - 编译入口代码（`main.cpp`）
  - 链接 `pi_guard_runtime`，生成 `pi-guard-agent`
